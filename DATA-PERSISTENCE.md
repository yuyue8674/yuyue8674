# GEO获客系统 - 数据持久化说明

## 数据存储改进

系统现已升级为**文件持久化存储**，所有数据会自动保存到本地文件中，不会因为页面刷新或服务重启而丢失。

## 数据文件位置

```
data/geo-data.json
```

此文件包含所有系统数据：
- 企业信息
- 关键词配置
- 地理围栏设置
- 搜索日志

## 功能特性

### ✅ 数据持久化
- 自动保存所有新增、修改、删除操作
- 服务重启后数据依然存在
- 无需配置数据库即可使用

### ✅ 预置演示数据
系统首次启动时会自动初始化3个演示企业：
1. **美味轩餐厅** - 川菜粤菜餐厅
2. **阳光健身房** - 24小时健身中心  
3. **咖啡时光** - 精品咖啡店

以及7个关键词和3个地理围栏配置。

### ✅ 实时保存
每次操作都会立即写入文件，确保数据安全：
- 添加企业 ✓
- 编辑企业信息 ✓
- 删除企业 ✓
- 配置关键词 ✓
- 设置地理围栏 ✓
- 搜索记录 ✓

## 使用说明

### 1. 添加企业
在企业管理页面，点击"添加企业"按钮，填写企业信息后保存，数据会立即持久化到文件。

### 2. 配置关键词
在关键词管理页面，选择企业并添加关键词，设置优先级和匹配类型，保存后数据会持久化。

### 3. 设置地理围栏
在地理围栏页面，为企业创建覆盖区域，数据会自动保存。

### 4. 测试搜索
在搜索演示页面，输入关键词测试搜索效果，搜索记录会被保存用于统计分析。

## 数据备份

建议定期备份 `data/geo-data.json` 文件，以防意外丢失。

## 技术实现

- 使用 Node.js fs 模块进行文件读写
- JSON 格式存储，易于查看和备份
- 单例模式确保数据一致性
- 自动创建数据目录

## 注意事项

- 数据文件会随使用逐渐增大，建议定期清理旧的搜索日志
- 如需重置数据，删除 `data/geo-data.json` 文件后重启服务即可恢复默认数据
